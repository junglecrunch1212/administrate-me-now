## C. DERIVED_COLUMNS ↔ always_derived equivalence — the named canary

Per `[BUILD.md §3.11]`, a forward sheet builder's derived-column set and the reverse descriptor's `always_derived` set must match exactly per (workbook, sheet). Drift means the principal can edit a backend-derived column and the reverse daemon will emit a spurious `*.updated` event for it (or treat the principal's edit as a deletion).

| Workbook | Sheet | Builder file & symbol | Builder derived set | Descriptor `always_derived` | Match |
|---|---|---|---|---|---|
| ops | Tasks | `sheets/tasks.py:36` `DERIVED_COLUMNS` | `{task_id, created_at, completed_at}` | `{task_id, created_at, completed_at}` | ✓ |
| ops | Commitments | `sheets/commitments.py:36` `DERIVED_COLUMNS` | `{confidence, strength, source_summary}` | `{confidence, strength, source_summary}` | ✓ |
| ops | Recurrences | `sheets/recurrences.py:30` `DERIVED_COLUMNS` | `{next_due, last_completed_at}` | `{next_due, last_completed_at}` | ✓ |
| finance | Raw Data | `sheets/raw_data.py:45` `ALWAYS_DERIVED` | `{txn_id, plaid_category}` | `{txn_id, plaid_category, is_manual}` | ✗ **MISMATCH** |

### Finding C-1 (non-critical, deferred)

**Drift:** `adminme/projections/xlsx_workbooks/sheets/raw_data.py:45` declares `ALWAYS_DERIVED = {"txn_id", "plaid_category"}` but `adminme/daemons/xlsx_sync/sheet_schemas.py:167` declares descriptor `always_derived = frozenset({"txn_id", "plaid_category", "is_manual"})`.

**Severity:** non-critical. Cosmetic cell-level protection drift only.
- The descriptor's `always_derived` already governs reverse-diff behavior. The reverse daemon will correctly drop principal edits to `is_manual` regardless of the builder-side protection state — diff core treats `is_manual` as a derived column on the diff side.
- The builder's `ALWAYS_DERIVED` controls cell-level workbook protection (the lock applied via `apply_row_protection(ws, i, HEADERS, locked_columns=locked)` at line 124). With `is_manual` absent from `ALWAYS_DERIVED`, the cell is not locked — but Raw Data has no live workbook-side edit path that targets `is_manual` (it's set by the projection from event type, not the principal). Bookkeeping inconsistency, not a correctness bug.

**Recommendation:** add `"is_manual"` to `ALWAYS_DERIVED` in `adminme/projections/xlsx_workbooks/sheets/raw_data.py:45`. One-line change. Ship as sidecar PR `sidecar-raw-data-is-manual-derived` per `E-session-protocol.md` "Single-purpose PR rule." Test addendum: assert `ALWAYS_DERIVED == descriptor_for(FINANCE_WORKBOOK_NAME, "Raw Data").always_derived` so the two stay in lockstep — that single assertion blocks any future drift in either direction.

### Runtime canary

`tests/integration/test_xlsx_roundtrip.py` is the runtime canary for any future drift of this kind: if a builder adds a derived column without updating the descriptor (or vice-versa), the next change to the involved column will start producing spurious `*.updated` events and the test's event-sequence assertions will fail. The audit formalizes this canary by name; no new test required at this checkpoint.
