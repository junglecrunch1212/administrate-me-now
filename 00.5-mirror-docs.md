# Prompt 00.5: Mirror external reference documentation

**Phase:** Pre-build, between prompt 00 (preflight) and prompt 01 (read artifacts).
**Depends on:** Prompt 00 passed. Repo is cloned; artifact files are present; environment verified.
**Estimated duration:** 1-2 hours (mostly WebFetch time + handling failures).
**Stop condition:** `docs/reference/` populated with every canonical external doc needed for the build; `docs/reference/_status.md` lists every URL with a ✓ (mirrored successfully) or ✗ (failed; manual clip needed by operator); `docs/reference/_refresh-schedule.md` exists.

---

## Why this prompt exists

External documentation sites (Plaid, BlueBubbles, Google, Apple, OpenClaw) often block bot traffic — Cloudflare, captcha walls, 403s to non-browser user agents. `WebFetch` sometimes works, sometimes doesn't. Relying on live documentation during the build means some prompts will fail to get the information they need, and Claude Code will either (a) fabricate answers from memory (which is often wrong because these APIs evolve) or (b) produce incomplete work and say "I couldn't fetch the docs."

The fix: mirror the canonical documentation into the repo once, as files. Every subsequent prompt reads from `docs/reference/` — no live fetches during the build. Cloudflare never enters the picture. Rate limits never matter. Documentation is version-pinned to whatever the mirror holds.

This prompt does the mirroring. For any URL that `WebFetch` can successfully retrieve, Claude Code writes a markdown file. For any URL that fails, Claude Code lists it in a "manual clipping checklist" that the operator completes in a normal browser (Chrome, with their real session, which no bot detector blocks).

---

1. `artifacts/ADMINISTRATEME_BUILD.md` — specifically the section **"BEFORE YOU START: LEARN OPENCLAW"** which lists OpenClaw URLs. Extract this list verbatim; it's the authoritative source for what OpenClaw docs to mirror.

## Operating context

You are Claude Code. Your job is to populate `docs/reference/` in the repo with mirrored copies of all external documentation the build will need. You do NOT understand or summarize the docs — you just save them as markdown files, preserving structure and source URLs. Later prompts read them.

For each URL:
1. Attempt `WebFetch`.
2. If successful: extract the main content (body; drop navigation/footer chrome); save as markdown in the appropriate subdirectory; prepend a header with source URL and fetch date.
3. If failed (4xx, 5xx, Cloudflare block, timeout): record in the failure list. Do NOT fabricate content.

At the end: produce a status report and a manual-clipping checklist for failures.

## Objective

Mirror all external reference documentation listed below into `docs/reference/`. Record successes and failures. Produce a schedule for future refreshes.

## Out of scope

- Do NOT mirror the AdministrateMe artifact files (they're already in the repo).
- Do NOT mirror anything not on the list below. Curation is important — this mirror is meant to be operationally useful, not comprehensive.
- Do NOT attempt to re-fetch URLs that fail. One attempt each; log the failure; move on. The operator handles failures manually.
- Do NOT try to be clever about circumventing Cloudflare or bot detection. If a site blocks you, record it and move on.
- Do NOT summarize, paraphrase, or reorganize the content. Mirror it verbatim. Trimming boilerplate (cookie banners, nav chrome, "was this page helpful?" footers) is fine; rewriting is not.

### For each URL

```
1. Determine filename: kebab-case the URL path. e.g.,
   https://plaid.com/docs/api/products/transactions/#transactionssync
   → plaid/transactions-sync.md
2. WebFetch the URL. If it returns 200 with substantive content: proceed.
3. Extract the main content. Strip nav, footer, cookie banners, "was this helpful?" blocks.
   Preserve: headings, body text, code blocks, tables, example payloads, links.
   (The links in mirrored docs may point to URLs you did NOT mirror — that's fine;
   the mirror is a snapshot, not a browsable site.)
4. Prepend the header (source URL, fetched date, purpose).
5. Write to docs/reference/<section>/<filename>.md.
6. On failure (4xx, 5xx, Cloudflare, timeout > 30s): add to failure list. Do NOT retry.
```

Priority rubric:
- **High:** OpenClaw (any), Plaid core flows (Link, transactions-sync, webhooks), BlueBubbles WebSocket events, Google Gmail push, Tailscale Serve — things cited by >2 build prompts.

## Verification

```bash
# All three status files exist and are non-empty
test -s docs/reference/_status.md
test -s docs/reference/_manual-clip-checklist.md
test -s docs/reference/_refresh-schedule.md

# Every section has an _index.md
for section in openclaw plaid bluebubbles google-gmail google-calendar \
               apple-eventkit apple-shortcuts tailscale caldav sqlcipher \
               sqlite-vec aiosqlite textual; do
  test -f "docs/reference/$section/_index.md" || echo "MISSING: $section/_index.md"
done

# Count mirrored files per section (expect at least 1 per, more in high-use sections)
for section in docs/reference/*/; do
  count=$(find "$section" -name '*.md' | wc -l)
  echo "$section: $count files"
done

# Every mirrored file has the required header
for file in $(find docs/reference -name '*.md' -not -name '_*'); do
  head -1 "$file" | grep -q '^# ' || echo "MISSING title heading: $file"
  head -5 "$file" | grep -q '^\*\*Source:' || echo "MISSING Source: $file"
  head -5 "$file" | grep -q '^\*\*Fetched:' || echo "MISSING Fetched: $file"
done

# Review the status report
cat docs/reference/_status.md
echo "--- FAILURES REQUIRING OPERATOR ATTENTION: ---"
cat docs/reference/_manual-clip-checklist.md

git add docs/reference/
git commit -m "phase 00.5: mirror external reference docs"
git push origin main
```
