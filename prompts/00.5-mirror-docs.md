# Prompt 00.5: Mirror external reference documentation (GitHub-first)

**Phase:** Pre-build, between prompt 00 (preflight) and prompt 01 (read artifacts).
**Depends on:** Prompt 00 passed.
**Estimated duration:** 45-75 minutes (mostly git clones + file copying).
**Stop condition:** `docs/reference/` populated with mirrored docs for every section; `docs/reference/_status.md` shows ≥90% coverage via GitHub; `docs/reference/_gaps.md` lists the small residual set (Apple, possibly Tailscale) needing manual clipping or allowlist expansion.

---

## Why this prompt exists

Your sandbox has an egress allowlist. `github.com` and `raw.githubusercontent.com` are allowed. Almost nothing else is — requests to `docs.openclaw.ai`, `plaid.com`, `tailscale.com`, `developer.apple.com` get `HTTP 403 x-deny-reason: host_not_allowed` from Anthropic's proxy before the request ever leaves the sandbox.

This blocks the naive approach ("WebFetch all external docs"), but it does NOT block the smarter approach: **most of the documentation we need lives as source markdown/code in public GitHub repos.** OpenClaw docs, BlueBubbles docs, Textual docs, Plaid's OpenAPI spec, Google's Discovery-generated API references — all on GitHub. Fetching those works within the sandbox today, with zero allowlist changes needed.

The GitHub-first pass gets you to roughly ~90% coverage. The remaining ~10% — Apple's EventKit/Shortcuts (Apple does not publish doc source anywhere), some Tailscale KB pages, and Plaid's narrative UX guides that aren't in the OpenAPI — get documented as gaps. The operator handles those by (a) asking to widen the allowlist for small well-behaved hosts like `rfc-editor.org` and `tailscale.com`, or (b) clipping the few Apple pages manually. Either way it's a short, finite task — not the 80-URL checklist an allowlist-unaware approach would produce.

## Operating context

You are Claude Code. Your job in this prompt is mechanical: clone or fetch from specific GitHub repos, extract specific files, and save them under `docs/reference/<section>/` with a header identifying the source. You do not summarize, paraphrase, or reorganize — you mirror verbatim.

You will hit allowlist denials if you try to reach non-GitHub hosts. That is expected. Those URLs are documented as gaps, not retried.

## Objective

Populate `docs/reference/` via GitHub. Produce a status report and a gap list. Commit and push.

## Out of scope

- Do NOT attempt WebFetch on any non-GitHub host. You will get 403s and waste time.
- Do NOT try to be clever about circumventing bot detection or allowlist rules. If a host is blocked, it stays blocked; that's a gap.
- Do NOT mirror the AdministrateMe artifact files (they're already in the repo).
- Do NOT summarize, paraphrase, or reorganize. Verbatim mirror.
- Do NOT include content from repos you haven't been told to fetch from. The list below is exhaustive.

### Sources to fetch (GitHub-first)

For each section below, the approach is: `git clone` (shallow, depth=1) into a tmpdir, copy the relevant files into `docs/reference/<section>/`, clean up the clone. Alternatively use `raw.githubusercontent.com` for individual files. Both work within the sandbox.

**OpenClaw** → `https://github.com/openclaw/openclaw` (mirror also at `openclaw/docs`)
- Copy everything from `docs/` subdirectory. Preserve subfolder structure if any.
- If OpenClaw's docs site has content not in the repo, note that in `_gaps.md`.

**Plaid** → `https://github.com/plaid/plaid-openapi`
- Copy the entire OpenAPI spec (probably `2020-09-14.yml` or similar named file + any referenced components).
- Additionally clone `https://github.com/plaid/plaid-python` and extract docstrings from the main API module(s); concatenate into `python-sdk-docstrings.md`.
- If any narrative markdown exists (`README.md`, `docs/`), copy that too.

**BlueBubbles** → `https://github.com/BlueBubblesApp/bluebubbles-docs`
- Copy `server/` subdirectory (GitBook source).
- Also check `README.md` of `BlueBubblesApp/bluebubbles-server` for any API summary.

**Google Gmail API** → `https://github.com/googleapis/google-api-nodejs-client`
- Fetch `src/apis/gmail/v1.ts` — single file with full docstrings for every endpoint.
- If a `README.md` specific to Gmail exists under `src/apis/gmail/`, include it.

**Google Calendar API** → same repo
- Fetch `src/apis/calendar/v3.ts` — same pattern.

**Textual** → `https://github.com/Textualize/textual`
- Copy everything from `docs/` subdirectory (MkDocs source, all markdown). Specifically need: `guide/app.md`, `guide/screens.md`, `guide/reactivity.md`, `guide/testing.md`, plus the landing `index.md`.

**aiosqlite** → `https://github.com/omnilib/aiosqlite`
- Copy `docs/` subdirectory (Sphinx .rst).
- Also copy `aiosqlite/core.py` (contains autodoc-ready docstrings that cover the API).

**sqlite-vec** → `https://github.com/asg017/sqlite-vec`
- Copy `README.md` and `ARCHITECTURE.md`.
- If `site/` or `docs/` exists, copy that too.

**SQLCipher** → `https://github.com/sqlcipher/sqlcipher`
- Copy `README.md`.
- Extract PRAGMA documentation from `src/pragma.c` — find the section documenting SQLCipher-specific PRAGMAs (key, rekey, cipher_*) and save to `pragma-source-excerpt.md`.

**CalDAV** → `https://github.com/python-caldav/caldav`
- Copy `README.md` (covers practical CalDAV client usage).
- Also grab `https://github.com/robur-coop/caldav/blob/master/README.md` if available — OCaml server with protocol notes.

**Apple EventKit, Apple Shortcuts, Tailscale, RFC 4791** → these are the GAPS. Create `_index.md` files in each section that document:
- Why the content isn't mirrored (not on GitHub, or Apple doesn't publish source).
- What the operator should do (specific URLs to clip manually, or the small set of hosts to ask Anthropic to allowlist).
- Priority (most Apple/Tailscale content is orientation, not API-truth — build can proceed with it missing; later prompts that need specifics will flag gaps).

### Manual clipping procedure

1. Open each URL in Chrome with your normal session.
2. Copy the main content (skip nav, footer, "was this helpful").
3. Paste into the target file with this header:
   ```
   # [page title]

   **Source:** <URL>
   **Fetched:** <today's date>
   **Method:** Manual clip (Apple docs not on GitHub)

   ---
   ```
4. Commit: `git add docs/reference/apple-eventkit/ && git commit -m "docs: manual clip of Apple EventKit docs"`
5. Push.

Optional: use a Chrome extension like "Markdown Web Clipper" for cleaner output.
