## Operating context

This is the final audit before the bootstrap wizard is built. Everything functional is in place: event log, schemas, all projections, skill runner, all pipelines, all adapters (standalone + plugin), all Python product APIs, the Node console, OpenClaw integration code, persona compiler, audit tool.

The purpose of this checkpoint is twofold:
1. **Re-verify every invariant from `SYSTEM_INVARIANTS.md`.** During the build, drift may have occurred. This is the last chance to catch architectural issues before the wizard is built on top of them.
2. **Confirm Phase B readiness.** The four bootstrap registration queues must be complete. When the operator runs `./bootstrap/install.sh` on the Mac Mini, these files tell the wizard what to install into OpenClaw. Missing entries = broken bootstrap.

## Objective

Produce `docs/checkpoint-15.5-report.md`. Every invariant from SYSTEM_INVARIANTS.md gets a PASS/FAIL. Every registration queue gets an inventory check.

## Out of scope

- Do NOT build the bootstrap wizard yet (prompt 16).
- Do NOT fix bugs found in this checkpoint — record and either fix in a follow-up session or defer.

## Deliverables

### The invariant re-verification

For each section in `docs/SYSTEM_INVARIANTS.md`, for each numbered invariant, run the appropriate check:

- **Grep-able invariants** ("no pipeline writes to projections," "every outbound goes through outbound()"): use grep or AST analysis to verify.
- **Schema invariants** ("every event type has a registered schema"): load the registry and enumerate.
- **Integration invariants** ("every skill call emits skill.call.recorded"): trace the code paths.
- **Runtime invariants** ("projection rebuild is deterministic"): run the relevant tests.

Produce a table: Section / Invariant # / Check result / Evidence or failure detail.

### Registration queue inventory

For each of the four bootstrap YAMLs, list every entry and cross-reference:

**`bootstrap/pack_install_order.yaml`** should contain:
- Every skill pack from `packs/skills/` (enumerate).
- Every pipeline pack from `packs/pipelines/` (enumerate).
- Every persona pack from `packs/personas/` (enumerate).
- Every profile pack placeholder (profiles are tenant-specific; configured during bootstrap).

**`bootstrap/plugin_install_order.yaml`** should contain:
- Every plugin in `adminme/openclaw_plugins/` (enumerate).

**`bootstrap/slash_command_registration.yaml`** should contain:
- Every slash command declared in any Python product API's slash-command registration code. The authoritative list is BUILD.md L5 (22 commands across 4 products).

**`bootstrap/standing_order_registration.yaml`** should contain:
- Every proactive pipeline from `packs/pipelines/` with `triggers.proactive=true` or `triggers.schedule` — the six from prompt 10c.

Any mismatch is a critical issue (the bootstrap wizard will either fail to install something or try to install something that doesn't exist).
